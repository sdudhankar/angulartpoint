Angular Dependency Injection



Dependency Injection (DI or IOC (inversion of control).


A coding pattern in which a class receives the instances of objects it needs (called dependencies) from an external source (angular injector) rather than creating them itself.
1. Build the Service: create service class.
How to generate service using CLI
ng generate service log
or
ng g s log
or
create new file log-interface

2. Registering the Service: Define Metadata with decorator.
Registering a service can we achieve in two way:

i) Register with component : child component can also access. 

@Component({{ '{' }}
selector: 'iva-app',
template: `Hi`,
providers:[EmpService]
{{ '}' }})

ii) Register with app.module: accessible in all application. 

@NgModule({{ '{' }}
providers: [EmpService],
{{ '}' }})
