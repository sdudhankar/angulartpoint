Angular Input and Output



1. @Input: @Input TypeScript supports decorators (sometimes referred to as “annotations”) which are used to declaratively add to or change an existing thing.
This annotation is a way to ask Angular to automatically bind the value of the input variable into the property variable.
To define an input for a component, we use the @Input decorator.
For example, our <user-profile> component needs a user argument to render information about that user:

  <user-profile [user]="currentUser"></user-profile>
So, we add an @Input binding to user:
 
import   {{ '{' }} Component, Input   {{ '}' }} from '@angular/core';
 
@Component({{ '{' }}
  selector: 'user-profile',
  template: '<div>{{ '{' }}{{ '{' }}user.name{{ '}' }}{{ '}' }}</div>'
  {{ '}' }})
export class UserProfile{{ '{' }}
  @Input() user;
  constructor(){{ '{' }}  {{ '}' }}
  {{ '}' }}
Note:Angular Recommended to use @Input() and @Output() Decorator. instead of input and output properties Because It is easy to read and identify. We can use private and public type also.
