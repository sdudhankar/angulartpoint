Angular Component



Angular is built upon components. Component is called as Heart of the Angular Development. The starting point of an Angular app is the bootstrapping of the initial parent component. Angular runs on a component tree model.

Components are classes that interact with the Template(.html file) of the component, which gets displayed on the browser. Component deals with the view of application and logic of the page.

What Component Contain?


As shown in above figure, Component is combination of Template, Class and Metadata.

1. Template:

It Defines the View of the Application. only Template is the place where we can write HTML code.
There are two ways to define Template in class.
i). template: Write html code in the component itself without creating separate file.


ii). templateUrl: Creating separate .html file and supplying it's url.

2. Class:

Model or prototype of an object that describe the properties and behavior of an object called class. inside class you can declare variables and method. class is a group of objects that has common properties.



3. Metadata:

Metadata is component decorator which added metadata and class follow it.

Decorator: A function that add metadata to a class, its member, or its method arguments. Prefix with @, Angular provide built in decorator like @Component , you can create your own also.
@Component: this is component decorator. always declare above the class, do not end with semicolon, decorator is a function so always add parenthesis (), we passed the object to the component function as an indicate with curly braces {{'{'}} {{'}'}}.

How to use style sheet in Angular Component?

A style sheet is useed to tells a browser how to render a page. it Defines HTML page color, font, position and generally where and how something is presented.

There are two ways to define styles insides the component.
i). styles:
Specifing styles in the component itself without creating separate file.
syntax: styles:[]

It uses square bracket ([]) to accepts multiple style sheets for single template.

ii). styleUrls:
Creating separate .css file and supplying it's url.

Create component with CLI

Sometimes we don't want to create basic structures over and over again. So @angular/cli has a solution. Using generate function of ng we can create components without any labor.

Step 1:

ng generate component <name>

or

ng g c <name>


for example, If we want to create Welcome component;

ng generate component welcome


Will generate welcome folder in app folder and It will create ts, html, css and .spec.ts files automaticly.
welcome.component.css
welcome.component.html
welcome.component.spec.ts
welcome.component.ts
welcome.module.ts

Also it will add declaration to app.module.ts. Basically it makes component ready to use.
