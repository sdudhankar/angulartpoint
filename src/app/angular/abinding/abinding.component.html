Binding



The communication/passing data between class and templates/view is called the Binding.

There are 4 types of binding in Angular



1. Interpolation or template binding:

Its binds variable to template.

  • Direction: one way binding. Template to view.
  • Symbol: {{ '{' }}{{ '{' }} {{ '}' }}{{ '}' }}
Syntax: {{ '{' }}{{ '{' }} data {{ '}' }}{{ '}' }}

2. Property binding:

Angular accept all property of HTML Tags and binds it with class. Like property of html tag (DOM). Property binding helps to control element property values from the component and change them whenever needed.

  • Direction: one way binding. Template to view.
  • Symbol: [ ]
Syntax: [property]="data"

There are 3 way to define property binding

Angular recommended to Property binding [ ] syntax for property binding.



3. Event binding:

Event is nothing but the action from the user like click, hover or any keyboard action.

  • Direction: one way binding. View to template.
  • Symbol: ( )
Syntax: (event)="statement"

Common types of Event Binding

  • (click)="myMethod()"
  • (dblclick)="myMethod()"
  • (submit)="myMethod()"
  • (scroll)="myMethod()"
  • (keypress)="myMethod()"




4. Two-way Binding:

The idea this is intended to invoke is that we're using both the input [] brackets and the output () parenthesis. It's an indication of the two-way bind. the [(ngModel)] is called a "banana in a box" syntax.

  • Direction: Two-way binding. Template to view and vise versa
  • Symbol: [( )]
Syntax: [(ngModel)]="property"
