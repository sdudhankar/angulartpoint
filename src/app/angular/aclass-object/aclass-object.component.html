Class and Objects in Angular



Class:

Class is a blueprint for creating objects. class helps to bind the variables and method. javascript and earlier version of ES5 doesn't support this feature. but typescript get this feature from ES6.

How to create class?


export keyword gives accessibility of other classes to use it and import the class.

A class definition can include the following −

1. Variables

−Any types of variable can declared inside a class.

2. Constructors

− Responsible for allocating memory for the objects of the class

3. Functions

− Functions represent actions an object can take. it also called methods.

1. Object creation must be outside of class.
2. All the javascript code can access inside the method only.

Constructor


When a class has no constructor defined explicitly one will be created automatically
In TypeScript you can have only one constructor per class.

Constructor overloading is not allowed in typescript so go for below constructor overloading.

In Angular2 constructors are used for dependency injection and inject services.
The associated decorator (@Component or Injectable) collects metadata (types or hints within @Inject) to determine what to provide to the object to instantiate.
constructors aren't part of the component lifecycle.

Access Specifier


1. Private: access inside the class only.
2. Public: access everywhere.
3. Protected: accessed by instances of deriving classes.
e.g. private test: string="it's a private variable";

Function or Method

1. Simple function
2. Function with return type
3. Parameterized function
4. Rest parameter
5. Default parameter
Return type of method is void but when methods do not declare an explicit returning type and return a value, it is assumed they can return anything (any type).
