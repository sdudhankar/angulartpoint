Project Setup



To install Angular, the Angular team came up with Angular CLI which eases the installation. You just need to run through a few commands to install Angular.

Angular CLI Installation step by step

step 1: Open Command Prompt

Press window button + R and type cmd


step 2: Go to the directory where you want to install application.

npm install -g @angular/cli


step 3: To create New Project with directory and files

ng new PROJECT-NAME


PROJECT-NAME is a name of your application.
step 4: Go to the project folder.

cd PROJECT-NAME


step 5: Deploy the Project

ng serve


step 6: Check the output

open the browser and type localhost:4200


If any other application is running on same port or you want to forcefully run the application on different port use the below command.
ng serve --port 1234
OR
ng serve -p 1234
Here 1234 is port number you can give any 4 digit number.

Investigating created project

Open your created project in Visual Studio IDE

The basic directory structure of the project

After opening project folder we can see e2e folder, node_modules, src folder, and some files. All our project source stuff are in src folder. Other files contains information about configurations, required packages etc. We will check details of all in Application Structure.

Inside of src folder there are multiple files. The most important file in this folder is index.html file. This file is the main point of our project. If you open it you will see something like this.

If we check above index file, we can see there is one special tag which aren't part of HTML standards. It's app-root element.
If you check app folder. Go dive into app.component.ts file.

As you can see there is some property selector app-root under @Component. In angular components will bind to element that given as selector. Selector is kind of like css element selector.
templateUrl property routes component's template location where you can write your HTML code. You can use template property too if you don't want to create html file.

styleUrls property shows us component's css file location. You can also use styles too if you don't want to create separate css file. Only the difference between template and styles, styles is expecting an array of string, not directly string. it means you can supply more than one styleUrls if you need it.




In angular projects works under modules. These modules look like java packages or c# namespaces. Your components defined in a module. app.module.ts file contains our main module.

All components that using in this project should declared in this file. If we don't declare it, angular couldn't find component that we want to use.
