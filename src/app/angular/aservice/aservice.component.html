Angular Service



Problem in Component
1. Code Duplication
2. Hard to maintain.
Comonent 1 doSomething()
Component 2 doExtra()
doSomething()
Component 3 doSomething()
for example. Component 1 have a function doSomething() whereas Component 2 has same function doSomething() and some additional function doExtra() and again Component 3 has doSomething() function. so unnecesarily we are making dublicate function for same work.

What will be the solution for this?
Centralized this task or function by using Service.

Service


"A class with a focused purpose."
"Logic/data which is not specific to view & want to share across component for that we build service."

Use for:
i) Are independent from any particular component.
ii) Provide shared data or logic across component
iii) Encapsulate external interactions such as data Access.

So shifting all this work from component to service it helps to test, debugged & reused.

Process to Create Service: (same as component)
1. Build the Service: create service class.
2. Registering the Service: Define Metadata with decorator.
3. Injecting the service: import what we need.

There are 2 way component can work with services
1. Creating instance of service class & use it.

i) Service

  export class myService
  {{ '{' }}
  {{ '}' }}
ii) Component

Let svc=new myService();

Drawback: instances are local to component & difficult to test.

2. Register service with Angular

• Here angular create single instance of that service class e.g. log, math, svc instance called singleton.
• specifically angular provide built in injector. We register service with angular injector, which maintain container of created service instance.
• if component, require service it use as a dependency.
• Angular provide/Inject service class instance when component class instantiated this process is called dependency injection.
